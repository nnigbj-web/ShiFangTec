1. 战略定位 / STRATEGIC POSITIONING

"吃什么吃" 为何能击溃传统饮食 App？

传统工具（如薄荷）死于"手动输入的繁琐感"。"吃什么吃"通过"一屏一对话"和"拍照即流式输入"实现交互范式的代际跃迁——这不是改进，是重构。

🎯

极致降低记录摩擦阻力

传统记录一顿饭需经历：搜索菜名 → 选择克数 → 手动加总，平均耗时 120秒。而"吃什么吃"的核心设计哲学是打开App即相机，将完整价值闭环压缩至 3-10秒。这是从"命令行"到"图形界面"级别的范式跃迁。

🧩

场景驱动而非功能堆砌

不强迫用户学习营养学。将能力封装成"Skill（技能卡片）"，通过直接场景（血糖负荷、新鲜度度量、热量等价代换）给予"就一件事"级别的行动方案。对应行为科学中的 BJ Fogg 模型：B = MAP（行为 = 动机 × 能力 × 提示），同时降低了能力门槛并提供了清晰提示。

⚡

渐进式档案填充 (DDF)

杜绝冷启动长表单。前3次免注册体验核心快判 → 第4次提示注册 → 第10次引导补充身高体重。参考 Duolingo / TikTok 的渐进式数据采集模式：先让用户体验核心价值，再逐步索取信息，大幅提升首次体验完成率。

2. 用户价值与地缘差异 / USER VALUE & GEOGRAPHY

三大地缘市场的痛点与 Skill 契合度

注：当前聚焦中国市场验证PMF，日本与欧美为长期愿景储备

🇨🇳 首发市场 · 高频外卖与复合中餐

中国市场：追求效率、隐形热量极多

中餐多属混合烹饪（红烧、煎炸、爆炒），富含隐形油糖，外卖心智极强。用户核心痛点：秒级获知"这顿外卖对我的控糖/减脂目标影响有多大"。

▶ 隐形热量曝光卡片 （揭示红烧肉、炒蛋里的隐形油糖）
▶ 极简平替建议 （"就一件事：加一份清炒时蔬"）
▶ 外卖场景全覆盖 （中餐复合菜的视觉识别专项训练）
💡 V1.0 核心突破点：餐盘快判 + 去幻觉精准数据 + 行动指南
🇯🇵 长期愿景 · 平衡膳食与高频陪伴

日本市场：一汁三菜、高信任度与 IP 陪伴

日本国民崇尚精致的"一汁三菜"均衡膳食，但面临微量元素缺乏问题。同时对注册营养师和虚拟 IP 陪伴有全球最高的信任与粘性度。

▶ 拟人化伴走师 （类似于 Asken 的"未来さん"式温情反馈）
▶ 每日微量平衡图谱 （一汁三菜精细化追踪）
💡 参考方向：Asken的IP伴走模式值得学习，但需自建日式食物数据库
🇺🇸/🇪🇺 长期愿景 · 极致Macros量化主义

欧美市场：精确数值、条形码扫码心智

欧美用户热衷于条形码扫描（UPC Scan），极度关注蛋白质、脂肪、碳水（Macros）的精确克数，常年践行生酮（Keto）、古法（Paleo）等严苛饮食法。

▶ 结构化 Macros 拆解 （极致精细的宏量元素百分比卡片）
▶ 饮食派系合规验证 （快速判定该食物是否满足 Keto 规范）
💡 参考方向：MyFitnessPal的Macros追踪 + Foodvisor的视觉识别路线

3.5 竞争格局深度分析 / COMPETITIVE LANDSCAPE

三层竞争模型：真正的威胁不是薄荷

仅将"吃什么吃"与垂类App比较是不完整的。真正的竞争是谁来成为用户饮食决策的第一触点。我们需要从三个层级审视竞争格局。

竞争三层次框架

第一层

直接竞品：垂类饮食App

薄荷健康、MyFitnessPal、Asken、Foodvisor —— 同一赛道的垂类玩家。这是最容易看到的竞争，但不是最危险的。

策略：交互范式代际跃迁（Web 2.0 → AI Native），用下一代产品打上一代产品。

第二层

平台巨头：真正的威胁

字节跳动（抖音/豆包）、美团、腾讯（微信/混元） —— 拥有流量、AI能力、用户数据和交易闭环的超级平台。这是最需要严肃对待的竞争对手。

策略：不在AI能力上竞争，在垂直数据深度 + 健康信任 + 商业立场上建立不对称优势。

第三层

替代性威胁：范式变革者

AI硬件（Apple Vision Pro/Meta智能眼镜实时食物识别）、GLP-1药物替代饮食管理需求、语音助手饮食能力进化。

策略：持续观察窗口，目前不紧迫但需在产品路线图中保留弹性。

第二层深度拆解：平台巨头的威胁与弱点

  • · 组织形态：大公司做垂直场景只能以"功能"而非"产品"出现

🎯 核心竞争策略

1. 不在AI能力上与字节竞争

在垂直数据深度和健康信任上建立不对称优势。字节不会为一个功能建立3000+子类训练数据库。

2. 让"不接广告"成为可验证的事实

公开算法逻辑、与三甲医院营养科联合发表研究、完成医疗器械认证——把营销文案变成竞争壁垒。

3. 将SFOOD数据库变成行业标准

开放API、参与制定"中式食物AI识别"行业标准、发表论文——成为基础设施而非一个App。

4. 时间窗口内建立不可逆优势

以中国市场为核心，在巨头重视之前完成PMF验证+数据积累+用户迁移成本建设。

4. 商业模式 / BUSINESS MODEL

三层商业架构：从C端流量到B2B数据资产

"吃什么吃"的最优商业模式不是C端订阅，而是以免费C端产品为数据引擎，驱动B2B数据服务+Premium订阅的双轮模型。

🏗️

B2B 数据服务（利润引擎）

真实场景下的食品消费趋势、地域偏好、品类增长——卖给食品公司、连锁餐厅

公共卫生指数

4.5 护城河分析 / COMPETITIVE MOAT

可防御性：巨头为什么不能轻易复制？

任何战略分析最终必须回答一个问题：如果字节跳动决定做一个同类产品，你需要多久会死？以下是护城河的诚实评估。

01

健康信任壁垒

护城河深度：⭐⭐⭐⭐

"不接食品广告、不卖竞价排名"——这不是营销文案，而是商业模式层面的结构性决策。字节/美团的商业模式决定了它们不可能提供无利益冲突的饮食建议。

防御逻辑：这不是功能差距，是商业模式的根本冲突。巨头无法"部分信任"——用户对健康建议的信任是二元的。
02

垂直数据壁垒

护城河深度：⭐⭐⭐⭐

SFOOD 3000+中式食物子类数据库 + 权威营养映射——这不是通用视觉AI能解决的。中餐复合烹饪的识别需要专门的训练数据。

防御逻辑：巨头不会为一个垂类场景建设3000+子类的训练数据库。它们的策略是通用AI覆盖——在食物这个高精度领域天然不够。
03

数据飞轮壁垒

护城河深度：⭐⭐⭐½

用户拍照 → 训练数据 → 模型更准 → 体验更好 → 更多用户。这是间接网络效应——弱于社交网络的直接网络效应，但随时间持续强化。

防御逻辑：巨头可以复制功能，但无法复制已积累的千万级真实场景训练数据。时间是最不可逾越的壁垒。

诚实评估：当前护城河状态

⚠️ 当前弱点：护城河高度尚不足以完全抵挡巨头。健康信任和数据飞轮都需要时间积累——V1.0阶段是最脆弱的窗口期。

✅ 强化路径：

  • · 将SFOOD数据库从私有资产变为行业标准（发表论文、制定标准、开放API）
  • · 建立"饮食信用分"体系——用户长期数据沉淀形成的个人健康画像，迁移成本极高
  • · 与硬件深度绑定——成为智能厨房设备的"AI操作系统"，让各品牌设备数据汇入平台
  • · 获取医疗级信任背书——三甲医院合作、二类医疗器械认证、中国营养学会认证

⚠️ 需要警惕的护城河幻觉：

  • ❌ "AI能力壁垒"——大模型能力是通用的，你没有独占的模型优势
  • ❌ "先发优势"——在巨头面前，先发优势持续不了几周
  • ❌ "功能丰富度"——巨头可以快速复制任何功能
  • ✅ 真正的壁垒 = 独占数据 + 用户信任 + 生态锁定 + 时间积累

4.8 风险管理 / RISK MATRIX

识别并缓解关键风险

严重 · 高概率

巨头入场竞争

字节/美团/腾讯利用现有AI能力和流量优势推出同类功能，以零边际成本覆盖你的目标用户。

缓解：不拼AI能力，拼垂直数据+健康信任+时间壁垒。在巨头重视前完成PMF验证和核心用户沉淀。

严重 · 中概率

AI分析错误导致的健康风险与法律责任

食物识别错误→营养数据偏差→用户依据错误建议做出有害健康决策→法律诉讼。

缓解：去幻觉架构（数据与LLM解耦）是核心防线。增加免责声明、建立错误反馈机制、获取专业责任保险。

严重 · 中概率

"免费"承诺与商业化的结构性矛盾

核心功能免费承诺不可逆，未来推出付费墙时遭遇用户抵制，转化率远低于预期。

缓解：V1.0在架构中预留免费/付费边界（feature flag），用"核心免费+深度付费"而非"突然收费"模式。参考Spotify的免费增值模型。

中等 · 中概率

用户隐私与数据合规

食物照片含地理位置/家庭成员等敏感信息；健康数据受《个人信息保护法》严格约束；跨境数据流动受限。

缓解：隐私设计（Privacy by Design）、数据最小化原则、用户授权透明化、聘请数据合规顾问。

中等 · 中概率

用户行为改变难度——留存率远低于预期

饮食管理类App的行业平均D30留存仅6-10%。即使交互体验大幅优化，改变用户记录习惯仍是巨大挑战。

缓解：设计习惯钩子（Hook Model——触发→行动→可变奖励→投入）、饭点推送、饮食连续记录天数（Streak机制）、周报推送。

低 · 低概率

GLP-1类药物替代饮食管理需求

司美格鲁肽等GLP-1药物的普及可能从根本上改变用户对饮食管理的需求模式。

缓解：短期（3-5年）饮食管理App与药物是互补关系而非替代——用药者仍需饮食管理。长期保持观察。

5. 底层技术方案 / DE-HALLUCINATION ARCHITECTURE

确定性数据解耦：消灭 AI 瞎编卡路里

大模型（LLM）天生具有计算能力不确定与"幻觉"倾向。在涉及生命健康的营养学领域，绝对不能允许大模型直接计算热量。这是整个产品方案中最有技术壁垒价值的设计决策。

Step 1

视觉标签解离层 (Vision Transformer)

通过多模态视觉大模型将照片中的物理餐盘解离成【菜品标签】与【估算克数】。只做"这是什么、大概多少量"，不涉及卡路里运算。

Step 2

拦截 LLM：确定性映射检索 (D-Query)

将解离出的结构化标签送入静态映射查询器。数据由中国食物成分表/USDA权威数据库计算得出，确保卡路里、Macros的数值 100% 精确与确定。LLM在此阶段完全被绕过。

Step 3

LLM 最终语言包装与对话演绎

大模型根据权威数据库返回的精确结果，生成最终极具温度感和行动引导力（"就一件事"）的卡片推荐文本。LLM只做它擅长的事——语言生成。

[1] 拍照输入: 📷 拍摄一盘 "红烧肉配米饭"
⬇️
[2] Vision Transformer: 识别 {'红烧肉': 150g, '米饭': 200g, '番茄炒蛋': 100g}
⬇️
[3] ⛔ 拦截：禁止 LLM 计算
避免 AI 随意捏造 680kcal 为 300kcal
⛔
⬇️
[4] 权威食物成分库 SQL/RAG 映射:
- 红烧肉 150g ➡ 717 kcal (库 ID: 1245，源自中国食物成分表)
- 米饭 200g ➡ 232 kcal (库 ID: 4101)
- 番茄炒蛋 100g ➡ 178 kcal (库 ID: 3821)
⬇️
[5] LLM 自然语言生成包装:
"75分！这顿红烧肉+番茄炒蛋+米饭，总计约680kcal。隐形脂肪偏高——酱汁和炒蛋用油较重。就一件事：下一顿加一份白灼时蔬，轻松到85分。"

5.5 V1.0 界面信息架构 / INTERFACE ARCHITECTURE

对话流 + 侧边栏 + 用户中心：三种模式各司其职

V1.0以极简对话流为唯一主界面，同时预留侧边栏与用户中心作为低频入口。三种模式互补而非互相替代。

Layer 1 · 主界面

对话流

唯一的日常交互界面。所有能力通过自然对话触发，AI生成结构化卡片。卡片点击可展开为固定页面交互的详情页。

▸ 拍照/文字 → AI分析卡片

▸ 卡片 → 点击进入结构化详情页
▸ 快捷Skill条 → 一键触发常用能力

解决"当下这一刻"的决策问题——极快、极简、无摩擦。

侧边栏 · 左上≡

会话与智能体管理

类似豆包的会话列表与智能体切换。管理对话历史，未来承载多智能体入口（减脂教练、家庭营养师、慢病管理师等）。

▸ 搜索对话历史
▸ 智能体列表与切换（当前·未来）
▸ 对话置顶/归档/删除

解决"我有很多对话/智能体，怎么找"的问题。

用户中心 · 右上👤

健康数据仪表盘

头像点击进入。承载需要结构化展示的健康数据——个人信息、饮食报告、趋势图表、冰箱库存、目标管理等。

▸ 个人信息（身高/体重/目标/过敏原）
▸ 饮食报告（周报/月报/趋势图表）
▸ 饮食历史（日历视图）

解决"我的身体数据、饮食趋势、目标进展如何"的问题。

设计哲学： 对话流解决"当下这一刻"的决策——极快、极简、无摩擦； 用户中心承载"时间维度"的认知——清晰、结构化、有洞察。 两种模式服务不同的用户心智状态，不矛盾。

6.5 免费/付费边界设计 / FREEMIUM BOUNDARY

V1.0隐性设计，V2.0显性开放

即使在V1.0全部免费的阶段，也必须在架构中清晰定义免费/付费的能力边界。用feature flag控制，让V2.0的商业化水到渠成而非突然袭击。

功能模块 免费版 (V1.0起) Premium (V2.0起) 设计考量
餐盘快判 每日N次 (N≈5-10) 无限次 N值需数据验证，确保轻度用户够用
新鲜度检测 完整功能 + 批量检测 + 库存关联 基础版不阉割核心价值
等热量对比 1次/天 + 分享水印 无限次 + 去水印高清图 分享即病毒传播，水印是免费获客
对话历史 最近7天 30天/90天/全部 历史数据是核心迁移成本
GL血糖追踪 预览（3次免费） 完整连续追踪 V2.0核心付费功能
虚拟营养师 基础问答 深度对话 + 情感伴走 深度对话是付费墙后的核心价值
家庭多成员 — 最多5名家庭成员 家庭场景=高LTV
数据导出 — PDF报告 + CSV导出 数据便携性是信任加分项
广告 零广告 零广告 信任不可分级
💡 核心原则： 免费体验必须足够好到让用户留下来，但不至于好到让用户不需要付费。参考 MyFitnessPal——免费版有广告且功能受限，但核心记录功能完整可用，用户愿意为更好的体验付费。我们比它们更强的点是：免费版也是零广告。

6.8 V1.0 北极星指标 / NORTH STAR METRICS

PMF是否成立？三个层级回答

V1.0的成败不取决于下载量——取决于是否验证了产品-市场匹配（PMF）。只有以下三个层级的指标同时达标，才证明PMF成立，才应该进入V2.0的规模化阶段。

🥇
核心指标

周活拍照分析次数

≥ 5次/周

每用户每周使用餐盘快判的平均次数。覆盖大部分正餐场景。

为什么是这个数：一周21餐，≥5次意味着用户至少对1/4的正餐使用了拍照分析。如果达不到，说明产品没有融入用户日常。
🥈
留存指标

D30 留存率

≥ 20%

首次使用后第30天仍活跃的用户占比。

为什么是这个数：行业平均6-10%。20%意味着产品体验有实质性的代际优势。低于15%说明"拍照即答案"的概念没有真正改变用户行为。
🥉
病毒指标

等热量对比分享率

≥ 15%

使用"等热量交换器"后点击分享的用户占比。

为什么是这个数：产品自带增长引擎是独立App生存的关键。15%分享率意味着具备自传播能力，可以降低对买量的依赖。

🚦 PMF 判定矩阵

🟥 未达PMF： 任何一个核心指标不达标 → 继续迭代产品，不进入规模化阶段
🟨 弱PMF： 1-2个指标达标 → 分析未达标指标的根本原因，定向优化后复评
🟩 强PMF： 全部三个指标达标 → PMF验证通过，启动V2.0规模化+Premium商业化
🟦 超级PMF： 全部超标30%+ → 产品具备成为平台级产品的潜力，加速融资和扩张
